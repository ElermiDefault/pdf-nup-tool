# Architecture

## 核心流程

1. FastAPI 托管前端静态文件和后端 API。
2. 前端上传 PDF，并请求后端渲染缩略图。
3. 用户选择一个或多个连续页段。
4. 用户为页段设置合并规则，例如 4 合 1。
5. 前端将规则提交给后端。
6. 后端用 PyMuPDF 重新排版生成输出 PDF。
7. 用户下载生成后的 PDF。

## 合并规则示例

```json
[
  {
    "start_page": 3,
    "end_page": 10,
    "layout": 4
  }
]
```

页码建议在前端显示为 1-based，后端内部处理时转换为 0-based。
